Project Chronicle + Replay Engine
Ключевая долгосрочная фича форка llama.vscode (Token-Saving Replay Agent)
1. Project Chronicle (Хроника Проекта)

Extension автоматически ведёт структурированную память всего жизненного цикла разработки проекта.
После каждой большой задачи (или по таймеру / по триггеру) агент делает Consolidation Checkpoint:
Коротко фиксирует, что было сделано.
Под какое окружение (Environment Profile) это было сделано.
Ключевые архитектурные решения, созданные компоненты, команды запуска и т.д.

Всё хранится локально внутри проекта (папка .replay/ или лёгкий SQLite + векторный индекс).

2. UI — вкладка Chronicle

Отдельная боковая панель в extension (по стилю GitLens / Timeline, но специально для AI-разработки).
Красивая хронология в удобном виде:

text2026-04-23  → Windows 11 + uv + PowerShell
   └─ Инициализация: FastAPI + PostgreSQL + Alembic

2026-04-24  → Добавлен Docker + docker-compose + nginx config

2026-04-25  → Переход на async endpoints + Redis caching
3. Replay Engine (главная killer-фича)

Кнопка «Replay under new environment» (один-два клика).
Пользователь пишет (или выбирает):«Я всё делал под Windows + uv. Теперь нужно адаптировать весь проект под Gentoo Linux + emerge + systemd на сервере.»
Агент берёт весь Chronicle (не сырой чат, а только чистые чекпоинты) → запускает полный replay.
Patcher автоматически адаптирует:
Все команды, пути, package manager
Dockerfile / docker-compose
Service files, systemd units
Скрипты запуска, CI/CD и т.д.

На выходе пользователь получает:
Красивый diff всего проекта
Список изменений
Кнопку «Применить все»


Почему это сильно

Решает боль, которой сейчас нет ни у одного extension’а: перенос проекта между окружениями без переписывания всего руками и без засорения контекста.
Использует уже существующие у нас фичи (Patcher + Consolidation Pass + Environment Profile).
Dual-model: Main-модель планирует логику, Patcher делает точную адаптацию.

Token-Saving Replay Agent — Полный сбор идей (24 апреля 2026)
1. Общая цель проекта

Создать один единственный VS Code extension (форк llama.vscode), который решает главную боль всех агентов: «модель не понимает реальное окружение пользователя».
Dual-model архитектура: Main Thinker (умная модель) + Patcher (быстрый специалист по окружению).
Главный результат: агент не засоряет контекст мусором про ОС/шелл/пакет-менеджер и не забывает предыдущие решения.

2. Уже реализованные фичи (наш текущий уровень)

Consolidation Pass (сжатие патчей в один чистый environment delta)
Отдельный выбор local/cloud для Patcher и Main модели
Inline-патчинг с undo + visual diff
Environment Profile (ручной + потенциально авто-сканирование)

3. Архитектура Cognitive Stack («Многослойный мозг»)
Основная идея: не роли, а разные уровни внимания и памяти, как в человеческом мозге.

4. Killer Features

Project Chronicle — структурированная долгосрочная память проекта (чекпоинты + хронология).
Replay Engine — «Переиграть весь проект под новое окружение» за 1–2 клика (Windows → Gentoo, uv → emerge и т.д.).
Patcher как Environment Guardian — автоматически исправляет всё под текущий профиль до того, как Main модель это запишет.
Self-improvement loop (Hermes Evolver) — агент сам себя улучшает, патчит свои промпты и придумывает новые навыки.
Dashboard «Сколько токенов сэкономлено» + визуальная история патчей.

5. User Flow (как это будет выглядеть)

Первый запуск → авто-создание Environment Profile.
Обычная работа в чате / Agent Mode → автоматически работает Cognitive Stack.
Боковая панель Chronicle — красивая хронология проекта.
Кнопка «Replay under new environment» — главная вау-фича.
Кнопка «Force Patcher» и «Run Self-Improvement».

6. Token-экономия и производительность

Один шаг Cognitive Stack ≈ 45–70k токенов (вместо 200–400k+ у обычных больших-контекстных агентов).
MVP-вариант (3 слоя) ≈ 25–40k токенов.
Цель: первая итерация чуть медленнее, но все последующие — в 2–3 раза быстрее за счёт отсутствия потери контекста и повторных объяснений.

7. План упрощения (чтобы не умереть при разработке)

MVP Cognitive Stack: Concept Keeper + Strategic Thinker + (Detail Grinder + Patcher)
Director на жёстких правилах (без отдельной модели)
Evolver запускается редко (каждые 8–15 шагов или раз в час)
Режим «Эконом» — часть слоёв на маленьких локальных моделях
